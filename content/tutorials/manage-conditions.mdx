# Tutorial: Manage Conditions

This tutorial covers condition management from creation through clearing.

**Time:** 15 minutes
**Level:** Intermediate
**Prerequisites:** Understanding of loan workflow

## What You'll Learn

- Understanding condition types
- Creating conditions
- Requesting documents
- Clearing conditions
- Waiving conditions

## Understanding Conditions

### What are Conditions?

Conditions are requirements set by underwriting that must be satisfied before loan approval.

### Condition Types

| Type | Abbreviation | When Cleared |
|------|--------------|--------------|
| Prior to Docs | PTD | Before CD generation |
| Prior to Funding | PTF | Before funding |
| Prior to Close | PTC | Before closing |

## Step 1: View Existing Conditions

1. Open loan detail page
2. Navigate to **Conditions** tab
3. Review conditions list

### Condition Information

Each condition shows:

- Title
- Type (PTD/PTF/PTC)
- Status
- Due date
- Assigned documents

## Step 2: Create a Condition

### From Template

1. Click **Add Condition**
2. Select **From Template**
3. Search for template (e.g., "paystub")
4. Select appropriate template
5. Modify if needed
6. Click **Save**

### Manual Creation

1. Click **Add Condition**
2. Select **Custom**
3. Enter details:
   - Title: "Most recent paystub"
   - Description: "Provide most recent paystub showing YTD earnings"
   - Type: PTD
   - Category: Income
   - Due Date: Set appropriate date
4. Click **Save**

<Callout type="tip">
Use templates when possible for consistency and speed.
</Callout>

## Step 3: Request Documents

### Link to Document Request

1. Open the condition
2. Click **Request Document**
3. Select document type
4. Add instructions for borrower
5. Click **Send Request**

### Borrower Receives Request

Borrower receives:

- Email notification
- Portal message
- Clear instructions

## Step 4: Review Uploaded Documents

### Notification

When borrower uploads:

1. Receive notification
2. View in **Documents** tab
3. Review document

### Link to Condition

1. Open condition
2. Click **Link Document**
3. Select uploaded document
4. Document now associated with condition

## Step 5: Clear Condition

### Review Document

Before clearing:

1. Open linked document
2. Verify it meets requirement
3. Check quality and completeness

### Clear the Condition

1. Open condition
2. Click **Clear**
3. Add clearing notes (required)
4. Confirm clearing
5. Status changes to "Cleared"

<Callout type="warning">
Once cleared, conditions cannot be un-cleared without manager override.
</Callout>

## Step 6: Handle Rejections

### When Document Doesn't Satisfy

If document doesn't meet requirements:

1. Open condition
2. Click **Reject**
3. Enter rejection reason
4. Click **Request Updated Document**

### Borrower Notified

Borrower receives:

- Rejection notification
- Reason for rejection
- Instructions for re-submission

## Step 7: Waiving Conditions

### When to Waive

Conditions may be waived when:

- Compensating factors exist
- Guideline exception applies
- Risk is acceptable

### Waive Process

1. Open condition
2. Click **Waive**
3. Enter justification
4. Submit for approval (if required)
5. Manager reviews and approves
6. Condition marked as waived

## Best Practices

### Creating Conditions

1. **Be specific** - Clear requirements
2. **Set realistic dates** - Allow time for response
3. **Use templates** - Ensure consistency
4. **Categorize correctly** - PTD vs PTF vs PTC

### Clearing Conditions

1. **Review thoroughly** - Verify requirements met
2. **Document clearly** - Add detailed notes
3. **Don't rush** - Accuracy matters
4. **Track aging** - Monitor past-due items

### Communication

1. **Set expectations** - Tell borrower what's needed
2. **Respond promptly** - Review quickly when received
3. **Be clear** - Explain rejections fully

## Summary

You learned how to:

- View and understand conditions
- Create conditions from templates and manually
- Request documents from borrowers
- Link documents to conditions
- Clear conditions with notes
- Handle document rejections
- Waive conditions when appropriate
