# Underwriter Guide

This guide covers underwriting workflow, decision-making, and condition management in JaroLoan.

## Your Dashboard

The Underwriter Dashboard displays:

- **Review Queue** - Loans awaiting review
- **My Decisions** - Recent decisions made
- **Conditions Pending** - Conditions awaiting review
- **Performance** - Decision metrics

## Review Queue

### Queue Management

Loans appear based on:

- Assignment rules
- Priority indicators
- Time in queue
- Product expertise

### Selecting Loans

Choose loans to review:

1. Sort by priority
2. Filter by type
3. Click to open
4. Mark as "In Review"

## Reviewing a Loan

### File Overview

Start with summary view:

- Loan parameters
- Borrower summary
- Property details
- AUS findings

### Credit Analysis

Review credit thoroughly:

1. **Scores** - All three bureaus
2. **Trade lines** - Payment history
3. **Inquiries** - Recent applications
4. **Public records** - Collections, judgments

### Income Analysis

Verify income calculation:

1. **Documentation** - W-2s, paystubs, tax returns
2. **Calculation** - Monthly qualifying income
3. **Stability** - Employment history
4. **Trends** - Income trajectory

### Asset Analysis

Review asset sufficiency:

1. **Sourcing** - Asset origin verified
2. **Seasoning** - Meeting requirements
3. **Large deposits** - Documented
4. **Reserves** - Post-closing funds

### Property Analysis

Evaluate collateral:

1. **Value** - Appraisal review
2. **Condition** - Property issues
3. **Title** - Commitment review
4. **Insurance** - Coverage adequate

## Making Decisions

### Approve

When loan meets all guidelines:

1. Click **Decision**
2. Select **Approve**
3. Add notes
4. Confirm decision

### Approve with Conditions

Most common outcome:

1. Click **Decision**
2. Select **Conditional Approval**
3. Create conditions (use templates)
4. Categorize: PTD, PTF, PTC
5. Confirm decision

### Suspend

When missing material information:

1. Click **Decision**
2. Select **Suspend**
3. Document deficiencies
4. Specify what's needed
5. Route back to processor

### Deny

When loan cannot be approved:

1. Click **Decision**
2. Select **Deny**
3. Select denial reasons
4. System generates adverse action
5. Confirm decision

## Condition Management

### Creating Conditions

Use templates for consistency:

1. Click **Add Condition**
2. Search templates
3. Select appropriate template
4. Customize if needed

### Reviewing Submissions

When conditions cleared by processor:

1. Open condition
2. Review linked document
3. Verify requirements met
4. Clear or reject

## AUS Integration

### Viewing Findings

1. Navigate to **AUS** tab
2. View DU/LP results
3. Review findings/messages
4. Import as conditions

### Re-Running AUS

When changes made:

1. Click **Resubmit**
2. Review updated findings
3. Update conditions

## Best Practices

1. **Consistent decisions** - Use guidelines
2. **Clear conditions** - Specific requirements
3. **Document rationale** - Explain decisions
4. **Timely review** - Meet SLAs
